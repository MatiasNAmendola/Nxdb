class  Updates
 Instances of this class should be placed around combined update operations. It's primary purpose is to prevent intermediate and premature database optimization which will greatly impact performance. Instances of this class may be nested. If an update operation (either directly or through a query) is applied while one or more instances of this class are active, database optimizations and reindexing will not be performed until the final instance of this class is disposed. Note that instances of this class do not affect the actual application of updates operations which are still applied immediatly regardless. More...
